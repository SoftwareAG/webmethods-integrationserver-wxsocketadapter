3.0 Usage Notes

-  The ADK provides a sample adapter package, WmSampleAdapter, which can be used as a model for developing adapters compatible with the webMethods Integration Server 6.5 or later. The Sample Adapter provides code to illustrate creating an adapter definition class and template classes for connections, adapter services, polling notifications, and listener notifications.

-  Also delivered with the Sample Adapter is a simulated back-end resource, Sample Server, to mimic a banking application. The Sample Server is delivered as a Java executable, configurable using a properties file.

-  For a complete description of the Sample Adapter and the Sample Server see Appendix D of the webMethods Adapter Development Kit 6.5 User's Guide in the <Integration_Server_directory>/doc/adk directory.

4.0 Enhancements and Fixes

1-QN4QR
Provide hotlinks to server classes delivered with the IS Javadocs.

The javadocs delivered with the ADK contain references to Integration Server classes. Because these references are not hotlinked, the user cannot easily access the associated documentation.

This issue has been resolved. In the ADK 6.5 javadocs, the references to Integration Server classes are hotlinked. However, in order for the hotlinks to work, the Integration Server documentation must be installed in the default directory <Integration_Server_directory>\doc\api. For more information about installing the Integration Server documentation, see the webMethods Installation Guide.

6.0 Globalization

The webMethods Adapter Development Kit conforms to the international standards of webMethods, which require: operation in any country, locale, or language configuration as listed in the webMethods Installation Guide; support for character encodings; and proper formatting, display, and validation of data (such as number and date formats). These features have been tested in this release and any known issues are listed below.

6.1 Localization and Certification

The webMethods Adapter Development Kit 6.5 provides support for developing and testing adapters that are internationalized and are localizable into different languages. Adapters created with this kit can include support for operation in any country, locale, or language. Support for character encodings and proper formatting, display, and validation of data (such as number and date formats) has been provided and adapters that are built using the webMethods Adapter Development Kit 6.5 have access to the webMethods Integration Platform's internal Unicode character set support, data types, and other international features and functionality.

The libraries, features, and capabilities of the Adapter Runtime and thus the functionality in the Adapter Development Kit have been tested extensively in non-English environments and with non-ASCII data, including complete testing in a Japanese environment. The Sample Adapter and simulated back-end resource have been tested in non-English locales and can be used as an example for coding resource bundles and other international features. 

6.2 Configuration

If you expect to display non-English data in your webMethods tools, make sure an appropriate font is installed in your operating environment. If you see hollow boxes or black squares where you expect to see non-English data, modify your JRE font.properties file to reference an appropriate font. For information on modifying your font.properties file, go to Sun's Java website. For instructions, see:

http://java.sun.com/j2se/1.4.2/docs/guide/intl/fontprop.html

http://www.unicode.org/help/display_problems.html

7.0 Documentation

For webMethods Adapter Development Kit 6.5 technical documentation, go to the <Integration_Server_directory>/doc/adk directory. The documentation is also available on request from the webMethods Advantage Web site. These manuals are provided as PDF files:

-  webMethods Adapter Development Kit Installation Guide Version 6.5

-  webMethods Adapter Development Kit User’s Guide Version 6.5

To view the documentation, you must have Adobe Acrobat Reader 4.0 or later.

For webMethods Adapter Development Kit 6.5 Javadocs, go to the <Integration_Server_directory>/doc/api/adk directory. To view the Adapter Development Kit Javadocs, open the index.html file. The Javadocs are HTML files that describe the public APIs (packages, classes, interfaces, and methods) provided with the webMethods Integration Server.
